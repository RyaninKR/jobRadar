# JobRadar - 채용 모니터링 대시보드 제품 설계서

> **버전:** 1.0.0
> **최종 수정:** 2026-03-21
> **상태:** MVP 개발 진행 중

---

## 1. 제품 비전

한국 주요 채용 플랫폼(원티드, 잡코리아, 사람인, 잡플래닛)의 공고를 실시간 수집·통합하고, 자연어 대화로 탐색·분석할 수 있는 **AI 에이전트 기반 채용 모니터링 대시보드**.

### 1.1 핵심 가치

| 가치 | 설명 |
|------|------|
| **통합 뷰** | 여러 플랫폼의 공고를 하나의 대시보드에서 탐색 |
| **자연어 인터페이스** | 필터 조작 없이 대화로 원하는 공고 검색·분석 |
| **프로액티브 인사이트** | 트렌드 변화, 매칭 공고 알림을 에이전트가 먼저 제안 |
| **중복 제거** | 동일 공고의 멀티 플랫폼 게시를 자동 감지·병합 |

## 4. 데이터 수집 파이프라인

### 4.1 플랫폼별 수집 전략

| 플랫폼 | 수집 방식 | 수집 주기 | 비고 |
|--------|----------|----------|------|
| **원티드** | Public API + 웹 크롤링 | 1시간 | API 우선, 부족분 크롤링 보완 |
| **잡코리아** | 웹 크롤링 (Playwright) | 2시간 | 동적 렌더링 필요 |
| **사람인** | Open API (공식) | 30분 | API 키 발급 필요 |
| **잡플래닛** | 웹 크롤링 (Playwright) | 2시간 | 리뷰 데이터 제외 (ToS) |

### 4.2 정규화 스키마

```typescript
interface JobPosting {
  id: string;                    // UUID
  source: 'wanted' | 'jobkorea' | 'saramin' | 'jobplanet';
  source_id: string;             // 원본 플랫폼 ID
  source_url: string;            // 원본 공고 링크

  title: string;
  company_name: string;
  company_id: string | null;     // FK → companies
  location_city: string;         // 시/도
  location_district: string;     // 구/군
  job_type: 'full-time' | 'contract' | 'intern' | 'freelance';
  experience_min: number | null; // 최소 경력 (년)
  experience_max: number | null; // 최대 경력 (년)
  salary_min: number | null;     // 최소 연봉 (만원)
  salary_max: number | null;     // 최대 연봉 (만원)

  categories: string[];          // 직무 카테고리 (정규화)
  skills: string[];              // 기술 스택 (정규화)
  industry: string | null;

  description: string;           // 원문
  description_embedding: number[]; // pgvector

  posted_at: Date;
  expires_at: Date | null;
  scraped_at: Date;
  is_active: boolean;
  deduplication_hash: string;    // company + title + location 해시
}
```

### 4.3 중복 제거 전략

- `company_name + title + location` 기반 해시 생성
- 유사도 80% 이상인 공고를 클러스터로 그룹핑
- 대표 공고 1건 + 출처 플랫폼 목록으로 머지 표시

## 5. AI 에이전트 시스템

### 5.1 에이전트 구조

```
┌────────────────────────────────────────────────────┐
│              Orchestrator Agent                      │
│  - 사용자 의도 분류 (검색/분석/설정/잡담)              │
│  - 적절한 하위 에이전트로 라우팅                       │
│  - 멀티턴 대화 컨텍스트 관리                          │
└──────┬──────────┬──────────────┬───────────────────┘
       ▼          ▼              ▼
┌──────────┐ ┌──────────┐ ┌──────────────┐
│  Search  │ │ Analysis │ │   Alert      │
│  Agent   │ │ Agent    │ │   Agent      │
│          │ │          │ │              │
│ • SQL 생성│ │ • 트렌드  │ │ • 조건 파싱   │
│ • 시맨틱  │ │ • 비교    │ │ • 스케줄 등록 │
│   검색   │ │ • 요약    │ │ • 알림 발송   │
└──────────┘ └──────────┘ └──────────────┘
```

### 5.2 에이전트 Tool 정의

| Tool | 설명 | 입력 |
|------|------|------|
| `search_jobs` | SQL + 벡터 검색 조합 | query, filters, limit |
| `get_job_detail` | 특정 공고 상세 반환 | job_id |
| `analyze_trend` | 기간별 트렌드 분석 | skill/category, period, filters |
| `compare_companies` | 기업 간 채용 현황 비교 | company_names[], metrics[] |
| `set_alert` | 조건부 알림 등록 | conditions, frequency, channel |
| `get_user_context` | 사용자 프로필·이력 조회 | user_id |

### 5.3 대화 예시

```
👤 "요즘 Next.js 쓰는 스타트업 채용 많아?"

🤖 [의도: 트렌드 분석] → Analysis Agent
   Tool: analyze_trend({ skill: "Next.js", companySize: "startup", period: "30d" })

   "최근 30일간 Next.js 관련 스타트업 채용 공고는 총 142건으로,
    전월 대비 23% 증가했습니다.

    주요 채용 기업: 토스(12건), 당근(8건), 버킷플레이스(6건)
    평균 제시 연봉: 5,200~7,800만원

    [차트: 월별 추이] [차트: 연봉 분포]

    알림을 설정해드릴까요?"
```

| 저장한 공고 | `/saved` | 북마크한 공고 목록 |

| **임베딩** | Voyage AI / OpenAI embeddings | 공고 시맨틱 검색 |
| **알림** | Supabase Edge Functions + Resend | 이메일/웹 푸시 |
| **배포** | Vercel (앱) + Railway/Fly.io (크롤러) | 크롤러 장시간 실행 필요 |
| **UI 컴포넌트** | shadcn/ui + Recharts | 일관된 디자인 시스템 + 차트 |

## 8. 에이전트 네이티브 설계 원칙

### 8.1 대화가 곧 인터페이스
- 검색, 필터, 알림 설정 모두 자연어로 가능
- 대시보드 UI는 에이전트 응답의 시각화 레이어
- 에이전트가 "이 차트를 보여줄까요?" → UI 컴포넌트 동적 렌더링

### 8.2 프로액티브 에이전트
- "관심 스택(React) 공고가 이번 주 30% 급증했어요"
- "저장한 공고 3건이 마감 임박합니다"
- "비슷한 포지션 평균 연봉이 지난달 대비 상승했어요"

### 8.3 컨텍스트 지속성
사용자별로 관심 직무, 보유 기술, 선호 지역, 연봉 범위, 검색 이력, 저장 공고, 알림 설정을 지속적으로 관리하여 개인화된 경험 제공.

## 9. MVP 스코프 (4주)

| 주차 | 마일스톤 | 상세 |
|------|---------|------|
| **1주** | 데이터 파이프라인 | 사람인 API + 원티드 크롤러, 정규화 스키마, Supabase 테이블 |
| **2주** | 기본 대시보드 | 공고 목록/검색/필터 UI, 페이지네이션 |
| **3주** | AI 에이전트 통합 | Claude tool use 기반 자연어 검색, 채팅 UI |
| **4주** | 알림 + 분석 | 조건부 알림, 기본 트렌드 차트, 중복 제거 |

### MVP 이후 확장 로드맵
- 잡코리아/잡플래닛 크롤러 추가
- 이력서 기반 자동 매칭
- 기업 리뷰/평판 데이터 통합
- 지원 추적 (Apply Tracker)
- 면접 준비 에이전트

## 11. 데이터베이스 스키마

### 테이블 구조

- `companies` — 기업 정보 (이름, 규모, 산업, 로고 URL)
- `job_postings` — 채용 공고 (정규화된 공통 스키마)
- `job_posting_sources` — 공고별 원본 출처 (다중 플랫폼 매핑)
- `job_skills` — 공고-기술스택 다대다 매핑
- `skills` — 기술 스택 마스터 (정규화된 이름)
- `user_profiles` — 사용자 프로필 (관심 직무, 기술, 선호 지역)
- `saved_jobs` — 북마크한 공고
- `alerts` — 알림 설정 (조건 JSON, 빈도, 채널)
- `chat_sessions` — AI 대화 세션
- `chat_messages` — 대화 메시지 (역할, 내용, 도구 호출)
